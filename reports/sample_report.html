Crítico CAPTCHA Not Enforced Server-Side
Por qué importa
Riesgo grave: explotable con poco esfuerzo. Arreglar de inmediato.
Qué es
The login endpoint accepts authentication requests without a valid recaptcha_v2 token. The CAPTCHA exists only in client-side JavaScript and provides no protection against automation.
Cómo arreglarlo
Validate the CAPTCHA token server-side on every auth request before processing credentials. Reject missing/invalid tokens with HTTP 400.
Evidencia
POST http://localhost:8080/login.php with 'g-recaptcha-response' removed → HTTP 200, no CAPTCHA error in response.

CVSS 9.1 · A07:2021 – Identification and Authentication Failures

Alto No Rate Limiting on Authentication Endpoint
Por qué importa
Riesgo alto: debería arreglarse pronto.
Qué es
The authentication endpoint does not enforce rate limiting, enabling brute-force.
Cómo arreglarlo
Rate-limit auth (e.g. 5 failed attempts/IP/min), return HTTP 429 with Retry-After, and consider progressive delays or CAPTCHA escalation.
Evidencia
20 sequential failed logins with no 429 or lockout.

CVSS 7.5 · A07:2021 – Identification and Authentication Failures

Alto Session ID Not Regenerated After Login (Session Fixation)
Por qué importa
Riesgo alto: debería arreglarse pronto.
Qué es
The session identifier stays the same before and after authentication, enabling session-fixation attacks.
Cómo arreglarlo
Issue a fresh session ID on every successful login (and on privilege change).
Evidencia
Session cookie 'PHPSESSID' kept the same value across login.
